# How to Contribute

Thanks to everyone pitching in to make the Core Keeper modding documentation and tools better. If you notice something confusing, want to flesh out a guide, or just have a smart idea to help other creators, we are always happy to see new contributions.

## Contributing on GitHub

We manage all contributions through **GitHub pull requests**, so head on over to the respective GitHub and start editing! (Links to the GitHub repos are below). &#x20;

Here is how it works:

1. **Visit our GitHub repo.** You'll see the entire documentation there, open for everyone.
2. **Create a fork.** Make your own copy of the repo so you can edit it freely. Press the button on the top right for this.<br>
3. **Make your changes**. Tweak or add whatever you think improves things. Keep them clear and focused so it's easy for us to review them.
4. **Open a pull request.** When you're ready, submit a pull request back to the main repo. Our team will review it, give feedback if needed, and merge it once everything looks good and ready.

Every contribution helps us keep the documentation accurate, clear, and actually useful for the entire modding community. So seriously, a big thank you for taking the time to support it!
